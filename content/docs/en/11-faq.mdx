# Frequently asked questions

---

## Installation and compatibility

**What Android versions does Mis Gastos support?**

The app requires Android 6.0 (Marshmallow) or higher. It's tested on Android 10, 11, 12, 13, and 14.

**Is Mis Gastos available on iOS?**

No. Mis Gastos is exclusively for Android. There's no iPhone or iPad version.

**Why does Android block the installation?**

Android blocks APKs that don't come from the Play Store by default. Enable "Install from unknown sources" for your browser or file manager in Settings > Security (or Settings > Apps > Install unknown apps, depending on the manufacturer).

**Does the app ask for special permissions?**

Only the permissions needed for it to function: storage access (for backup) and internet (for OTA updates, only when you initiate it). It doesn't ask for contacts, camera, location, or notification permissions.

---

## Data and privacy

**Is my data uploaded to any server?**

No. Mis Gastos has no backend or user account. All data lives in your phone's local storage. The app makes no network requests except when you manually check for updates.

**What happens to my data if I uninstall the app?**

It's deleted along with the app. Android removes application data on uninstall. **Back up before uninstalling**.

**Can I view the backup file the app generates?**

Yes. The backup is a plain-text JSON file. You can open it in any text editor to view (or edit) your data.

**Do backups sync automatically to the cloud?**

No. The backup export opens Android's share sheet, where you choose where to save it (Google Drive, Dropbox, email, etc.). There's no automatic sync.

## Accounts and transactions

**Can I change the currency of an account?**

Not directly if it already has transactions — changing it would distort all historical amounts. The recommended solution is to create a new account in the correct currency and transfer the balance.

**Are there automatic recurring transactions?**

No. All transactions are logged manually. There's no option to automatically repeat a monthly expense.

**Can I import transactions from a CSV or Excel file?**

Not currently. The only supported import format is the JSON backup generated by the app itself.

**Is there a transaction limit?**

No limit enforced by the app. The only practical limit is device storage, but a JSON file with 50,000 transactions weighs less than 50 MB.

**Can I attach photos or receipts to a transaction?**

No. The "note" field is free text but doesn't support file attachments or images.

## Reports and categories

**Why don't some transactions appear in the charts?**

The transactions probably don't have a category assigned, or the active period/account filter excludes them. Check that the period filter covers the transaction dates.

**Can I have subcategories?**

No. The category hierarchy is flat — no subcategories or groupings. If you need more detail, use descriptive names (e.g., "Health - Pharmacy", "Health - Doctor").

**Do reports update in real time?**

Yes. Charts recalculate every time you open the Reports screen or change a filter.

---

## Debts and investments

**Can I track debts in different currencies?**

Yes. Each debt has its own currency. The summary totals on the debts panel convert everything to the base currency.

**Does the app connect to stock exchanges to update share prices?**

No. Investment prices are always manual. You update the current value whenever you want.

**Can I record a stock position with multiple lots at different prices?**

Not directly. An investment is a single position. You can record each purchase as a "Deposit" movement, which accumulates the total cost, but you can't see the average price per lot.

## Updates

**How do I know there's a new version?**

You check manually from **More > Updates**. The app doesn't send automatic update notifications.

**Can I update without losing my data?**

Yes. An update installed over the same app preserves all data. Still, it's always a good practice to back up first.

---

## Common issues

**The app crashes unexpectedly on launch**

Try uninstalling and reinstalling the APK. If you have a backup, import it afterwards. If the problem persists with a specific APK version, verify that your Android version is compatible.

**The backup doesn't import correctly**

Make sure the file is the JSON generated by the app (not a manually edited file with syntax errors). The JSON must have the root object with the standard keys: `accounts`, `transactions`, etc.

**An account balance doesn't match what I expect**

The balance is `initial balance + Σ income − Σ expenses ± transfers`. Check the account's transaction history by filtering by that account in Transactions to identify unexpected entries.
